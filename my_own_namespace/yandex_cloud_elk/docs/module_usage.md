Подробная документация модуля my_own_module

Описание
Модуль my_own_module предназначен для создания текстовых файлов на целевых хостах. Модуль идемпотентен — повторный запуск с теми же параметрами не изменит файл.


Параметры
path (обязательный)

Абсолютный путь к файлу. Если директория не существует, она будет создана автоматически с правами 755.


content (обязательный)


Содержимое для записи в файл. Поддерживает многострочный текст.

Возвращаемые значения

{
    "changed": true,
    "path": "/tmp/example.txt",
    "content": "Hello",
    "uid": 1000,
    "gid": 1000,
    "owner": "username",
    "group": "groupname",
    "mode": "0664",
    "size": 5
}
Примеры
- name: Create file
  my_own_namespace.yandex_cloud_elk.my_own_module:
    path: "/tmp/test.txt"
    content: "Hello Ansible"
Использование переменных

- name: Create file with variables
  my_own_namespace.yandex_cloud_elk.my_own_module:
    path: "{{ app_config_path }}"
    content: "{{ lookup('template', 'config.j2') }}"
Идемпотентность
Модуль проверяет существование файла и его содержимое. Если файл существует и содержимое совпадает:


changed: false
Файл не перезаписывается



Если файл не существует или содержимое отличается:

changed: true



Файл создаётся или перезаписывается

Check mode

Модуль поддерживает режим проверки (--check). В этом режиме:


Файл не создаётся


Возвращается changed: true если файл должен быть создан


Возвращается changed: false если файл уже существует с правильным содержимым
